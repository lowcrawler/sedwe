About SEDWE

The Sediment Data and Information Web-based (SEDWE) is created by the United States Geological Survey.

SED_WE provides users a range of hand-held platforms to handle and transfer USGS sediment-sample data and information collected in the field into the National Water Information System (NWIS), and to the USGS Sediment Laboratory Environmental Database System (SLEDS), through the Sediment-sample (Sed) application.

SedWE User Manual

The SedWE Field Application for USGS Sediment Data

SedWE is a field application for sediment data collection and entry for the USGS. SedWE can be used only at stations that are registered in the SedWE database, and only by people who are registered as users in SedWE and assigned particular stations to work at. Each station in SedWE is assigned specific email recipients, who receive, via email, field data submitted by the user of the SedWE field application.

As many of the field sites may be remote and internet access is not available, the SedWE application can be used in both on-line and off-line modes. In on-line mode, data are stored on the device and also backed-up across the network to a database (the data are 'saved' only, not submitted to SedLOGIN). When a user is working remotely in off-line mode, or without internet connection, data are stored on the device and cannot be saved to the network database until later.  (If the device is formatted or lost at this point, the data are lost). When a user has access to the internet later, the data may be backed-up to the network.  

[ Note:  Observers are not supported in Version 1.0 of SedWE:
SedWE makes it easy for non-USGS observers to enter basic data associated with the sediment samples an observer collects, and for USGS hydrographers (technicians) to enter more complete data associated with sediment samples. (Observers have less extensive powers than USGS hydrographers.) ]

SedWE also makes it easy to transfer the data electronically into the SedLOGIN application, for submission to a USGS sediment laboratory and to the QWDATA database in NWIS.

Note: SedWE requires an HTML5 compliant web browser with javascript enabled. Windows Mobile does not support HTML5. Additionally, SedWE does not work properly on Internet Explorer. Users should use Chrome, Firefox, Safari, Opera, Edge or another HTML5 compliant web browser.

OVERVIEW OF SedWE ARCHITECTURE

SedWE runs on hand-held devices, tablets, and laptops, but the software is web-based. Therefore, there is nothing to install on the device. All the user must do is point the device's browser to the SedWE website, and it downloads the software into local storage on the device. As long as the local storage is not cleared, the software exists on the device, and can still run, even when there is no longer an internet connection (which is the case at many field sites). Thus, SedWE can run in on-line mode (with an internet connection), as well as in off-line mode (when there is no internet connection).

Sediment data can be entered into SedWE similarly, in on-line mode, and in off-line mode. When in on-line mode, the data entered into the device are backed-up to the SedWE database when the user synchronizes with the SedWE website. This cross-network storage is called "synchronization", and we say the device is "synched" with the database.

When in off-line mode, the data entered into SedWE are stored only in the device's local storage. At a later time, when the device can be "on-line", the user can synch the locally stored data with the SedWE database. The data are not 'submitted' when the device is synched, rather the data are stored on the SedWE network.

A detailed description of the main menu is given hereafter, yet a brief overview is warranted at this point. The main menu contains selectable buttons: Shipments Manager, Add sets or groups of samples, Current Shipment, Review/History, Sync Device to Database, and Help. Each of these are described in detail later in this manual. In the upper right corner of the main menu is a circle with three horizontally stacked dashes, affectionately called a 'hamburger menu'. Clicking on this button reveals additional sections of SedWE: Main Menu, User Manual, and Clear Local Storage.

The Main Menu, and User Manual are mostly self-explanatory, and are explained in detail later, but the option to Clear Local Storage requires immediate explanation.  At times, a user may find it necessary to clear their local storage to start over with no data in the local device. Selecting the 'Clear Local Storage' option will allow that. Warning: if you have been working in offline mode and have not synched the SedWE application (see description of on-line/off-line modes described above) and select 'Clear Local Storage', all of the data entered will be lost.

OVERVIEW OF SedWE WORKFLOWS

The main work of SedWE is done in the SedWE client, where field data is entered; therefore, the majority of this document describes the workings of the SedWE client. What happens in the SedWE client is controlled by how users and stations are administered in SedWE, which is done in the SedWE Admin Module. A brief description of the Admin Module follows.

Admin Module

Each station registered in SedWE is associated with one state (such as Oregon, or Puerto Rico). Also, each station in SedWE is associated with one or more "email recipients" who receive the emails containing sediment data submitted by a SedWE user who has submitted data for that station.

Each user registered in SedWE is assigned one or more SedWE stations for which they may use SedWE. SedWE will not allow a user to enter data for a station NOT assigned to that user. Users are added to SedWE via the Admin Module. Adding a new user to SedWE, whether by the User Management screen or by importing from a SedLOGIN project, is a two-step process. The added user is first sent an email, notifying that he/she has been added and requiring the new user to verify by clicking on a verification link in the email. This verifies that the correct email address was entered for the user. When the user clicks on the verification link, a second email is sent, which completes the process of adding the user to SedWE. Until the second email is sent, the user is NOT YET a user of SedWE.

Control of the stations and users is done in the administration module of SedWE. All hydrographers registered in SedWE are allowed to register users and stations for sediment work with SedWE. Hydrographers may also import Users and/or Stations from their projects in SedLOGIN, into the SedWE admin database.

See the Admin Module's documentation, available in the Admin Module itself, for more details on registering users and station.

Field Data Workflow

In the SedWE application, as in SedLOGIN, the term 'sample' is equivalent to a unique record number in QWDATA. Thus, if a composite of 5 bottles were collected, those 5 bottles become one 'sample' that is entered into SedWE. Alternatively, if a cross-section of 5 bottles was collected and each bottle was to be analyzed individually, the user would log in five individual 'samples' into SedWE.

A brief outline of the workflow for field data is given here. The remainder of this document fills in the details.

The data structure of SedWE is patterned after the data structure of SedLOGIN, which is documented at (https://qwdx.cr.usgs.gov/uo/labxfer/help/sedlog_help.html). Using SedWE, the user enters sediment data as an "Event" of either one "Group" (limited to 40 or fewer single container samples per group), or one or more "Sets", similar to the events, sets, and groups in SedLOGIN. Instead of entering sample details in SedLOGIN, the user can now enter them in SedWE, which are later imported into SedLOGIN.

After entering the details of each sample in an event, the user submits the event from SedWE, and an XML file containing the sediment data is sent, attached to an email, to all the email recipients registered to receive submitted data for that station. The XML file is generated with the name: siteID_userID_YYYY-MM-DD.xml, where siteID is the 8 – 15 digit site ID, userID is the username of the person submitting the XML file, and YYYY-MM-DD is the date of the first sample entered.

[ note:  Observers are not supported in Version 1.0 of SedWE:
The SedWE client functions differently for a USGS technician than it does for an observer. A submission by an observer is not yet complete. The observer can enter only a few basic field parameters, so one of the recipients of the email must review the observer's submission, and specify the analyses, sampling methods, and sampler types. The technician who oversees the observer is expected to know what the observer is using. This reviewing person is also a USGS user, and is most likely working "on-line" within the USGS network. ]

Submissions by a USGS technician should also be reviewed to ensure completeness and accuracy. Instructions on how to open an XML file for review in SedWE is described in the "Reviewing Data" section below.

When a reviewer is satisfied that an event's data are complete and accurate, the reviewer saves the completed data to another XML file, which can be imported into SedLOGIN, providing one more interface to view/review the data, modify it if necessary, and submit it to the sediment laboratory and to QWDATA in NWIS.

MAIN MENU

Shipments Manager

The Shipments Manager menu is where sample information is entered, reviewed, edited, and submitted. The Shipments Manager is divided into three submenus:

  1. Add sets or groups of samples,
  2. Current Shipment, and
  3. Review/History.


Add sets or groups of samples is the menu which begins the process of entering data for suspended sediment, bottom material, and bedload samples. Current shipment is the menu in which data are viewed, modified, submitted, or deleted. The review/history menu lists all of the submitted events by a user. These menus will be described in detail.

Sync Device to Database

Selecting 'Sync Device to Database' effectively backs up the data from the device's local storage to SedWE's network backup database. These data have not been submitted for review or merged into SedLOGIN.  Synchronizing the device with the backup database occurs in three steps:

  1. First, any data marked for deletion on the device is deleted from SedWE's network backup database. (Actually, it is simply marked as deleted, but remains stored.)
  2. Next, any new data on the device, that does NOT already exist in SedWE's backup database, is copied to the database.
  3. Lastly, any data existing in SedWE's backup database which does NOT currently exist on the device, is copied to the device's local storage.

Help

The Help menu is where information regarding SedWE is found. The Help menu contains three submenus:

  1. Users Manual,
  2. About SedWE, and
  3. Release Notes.

The Users Manual describes how to navigate the application, the About SedWE section contains the SedWE version number, release date, contact email, as well as a brief description of SedWE, and the Release Notes section documents changes made with each version of SedWE.

MANAGING DATA in the SedWE CLIENT

Adding data

Adding data can be done in off-line and on-line modes. In the main menu, verify that the Shipments Manager submenus are expanded (this is the default viewing when in the main menu). Selecting the Shipments Manager line, or plus/minus icon at the right end of the line will expand or collapse the subheadings below. To add sample information, choose 'Add sets or groups of samples'.

The next screen, 'Create a new event', lists three available sampling methods: 1) Suspended Sediment, 2) Bottom Material, and 3) Bedload. Select the sampling method. The title of the next screen will depend on the selected method for the samples, yet the data entry options are the same. Enter the begin date for the event, and select other event options, as described in the SedLOGIN user's manual. You can also enter any miscellaneous comments for this sampling event, which can be useful as part of the site visit's record, and can be useful when working the sediment record later.  These event-level comments are not stored in NWIS, but will be stored with the sediment data in the XML file, for reference later, such as when working records.  This page is where you specify whether the samples are in a group, or whether they are in one or more set. When finished, select 'Add group or set' at the bottom right corner. The 'Back' button at the bottom left corner returns the user to the previous screen.

If the samples are one or more sets, the next screen is titled 'Modify Event' and lists the sets (Set A, Set B, etc.), however, if the samples are a group of single-container samples, this page will be skipped. For sets, select a set to begin entering data. The next screen is titled with the number of containers in this set, and lists the available analyses to be requested for the samples in the set. Select the desired analyses and select 'Add Sample Info' at the bottom right corner, to enter the screens to add detailed information for each sample record. You can also enter any miscellaneous comments for this set, or group of samples.  Such comments can be useful as part of the site visit's record, and can be useful when working the sediment record later.  These set- or group-level comments are not stored in NWIS, but will be stored with the sediment data in the XML file, for reference later, such as when working records. For example, comments in the 'Event Comment' may include sampling-event-related comments that will aid in the records working process, such as local conditions in the stream particular to your site, whereas a comment in the 'Set/Group' comment box relates more to issues particular for the set or group such as volume issues with a set of ISCO bottles, or high amounts of organic matter observed in a cross section set.

The next screen is titled with the type of sample and container for which the data are being entered. The data entry options on this screen are described in the SedLOGIN user's manual. Some fields are required, while others are optional. On this screen you can enter any sample-related comments, which will be stored in NWIS as the Sample Field Comment in QWDATA.  For example, in a bedload cross section, you must record the time the sampler spent on the bottom, for of EACH point in the cross section. And if you composite the cross section, there will be only one sample record for the cross section, so you probably want to store all the times in this one sample record's Field Comment. When complete, select 'Next Sample' at the bottom right corner of the page to advance to the next sample in the set or group. After selecting 'Next Sample' on the final sample in a set or group, the 'Modify Event' menu appears and displays the sets again. If another set has not had sample data added, select the set to be entered and enter the desired sample data. When all of the sets and samples have been completed, the 'Done' button appears. Selecting 'Done' returns to the Main Menu.

Adding data for an Observer

[ note:  Observers are not supported in Version 1.0 of SedWE:
When an observer adds data, SedWE allows entry of fewer fields. The observer enters the station, the sediment type, and the date and time, and only two field parameters: water temperature (P00010) and gage height (P00065). Observers are not expected to know all the options for sample-level fields of QWDATA (such as hydrologic condition, hydrologic event, sample type, etc.), or many of the field parameters that a technician knows about. For this reason, data collected by an observer MUST be reviewed and completed by a USGS reviewer, after the observer submits it. ]

Editing data

Editing data can be done in off-line and on-line modes. To edit sets or groups of samples that have been created, select 'Current Shipment' from the Main Menu. The 'Current Shipment' menu displays all of the events that have been entered by the user that have not yet been shipped. Each event has three icons below the event information: 1) a view icon (symbol of an eye), 2) a submit button (symbol of a letter), and 3) a delete button (symbol of an X). To edit the information, select the view icon. The next screen is the 'Modify Event' screen and displays the editable event information. For additional editing of the data, select the edit icon (symbol of a pencil) at the bottom of the screen next to the view and delete icons.

It is worth noting that a user cannot delete the last sample in a set or group, as doing so would result in a set or group without a sample. Instead, delete the entire group or event and start over.

Submitting data

Submitting data requires an internet connection (on-line mode only). Once the data are ready for shipping, select 'Current Shipment' from the Main Menu and locate the submit icon (symbol of a letter). Once the submit icon is selected, a pop up authentication window appears and requires entry of the username/email and the active directory password (for a technician), or the username/email and SedWE code (for an observer) before the data are submitted. Once the data are submitted, an email with the XML file will be sent to the user and all the email recipients of data for that SedWE station.

Reviewing data

To review the events that you have previously submitted yourself, select 'Review/History' from the Main Menu. The next screen displays the 'History & Review' options. To review a previously submitted event in the list, select the view icon (symbol of an eye).

To review an XML file submitted by someone else, you must pull that XML file into your own SedWE "Current Shipment" section. This can be done by saving that XML attachment (from the email you received) to your computer disk, enter SedWE and go to the Main Menu screen, select the 'Upload XML File into SedWE' button, and specify the XML file on your disk.

[ note:  Observers are not supported in Version 1.0 of SedWE:
Remember, submissions by observers MUST be reviewed, since the observer was not allowed to submit a complete dataset, such as specifying analyses, or several of the sample-level fields of QWDATA. ]

Also, initial submissions from USGS technicians should be reviewed by someone, to ensure completeness and accuracy of the data.

SedWE has an option for viewing sample data, which is helpful for viewing site visit data, for data review, and for the process of working sediment records. To view a sample report, go to History & Review, click on the View/Modify event button of the desired event, then click on the circle button with the icon of the eye on it, for each sample you want to review. This will open a sample report which contains details of that sample, as well as the event and set details. Any event comments, set comments, or container comments entered with this sample are shown in this sample report.

Re-submitting data

Re-submitting data requires an internet connection (on-line mode only). Whenever any changes need to be made during review of a submitted XML data file, the reviewer must re-submit the data via SedWE, to generate a corrected XML file. This corrected XML file is the one to load into SedLOGIN.

IMPORTING DATA INTO SedLOGIN

Importing an XML file into SedLOGIN can be done only by a user who is logged into SedLOGIN, and only into a SedLOGIN project that the user is a member of. SedLOGIN version1.8 and higher has an option to "Import Data from SedWE".

The import is done in 2 steps. First, the user specifies an XML file from his or her computer disk, and SedLOGIN reads that file and converts the XML format to the usual "wide" SedLOGIN display. The user can see the display, and decide if it looks correct BEFORE actually saving the data into SedWE.

When the user decides to save the data, the imported event's data are added to the project's "Home and Current Shipment". From this point, the user can still use the "Modify Mode" of SedLOGIN to correct any last-minute errors.

The user can submit the SedLOGIN's current shipment whenever ready. Perhaps the user may import more XML datasets from SedWE, or perhaps add more sets or groups via the SedLOGIN interface. In any case, when the user submits via SedLOGIN, those data are final (just as SedLOGIN's "submit" has always been). The data submitted via SedLOGIN are sent to the USGS sediment lab associated with that SedLOGIN project, and they are also uploaded into QWDX as field data for transmittal to QWDATA.

SedWE Release Notes

SedWE Version 1.0, released August, 2015

The first release of SedWE, the Sediment Data Field Application, provides a basic set of functions allowing a user to store field data at sediment sites for later transfer to SedLOGIN, as well as store miscellaneous field information for later use while working records. SedWE runs in web browsers such as Chrome, Safari, and Firefox (but not in Internet Explorer), and on labtops, iphones, and tablets (but not older technology, running Windows Mobile 6).

SedWE Version 1.0 consists of a client that enables the entry of field data, an administrative module that enables the setup of users and stations, a user’s manual that describes the functions of SedWE, and a central database that backs up and archives the datasets submitted from field devices running SedWE. A field device or computer accesses SedWE software over the internet, which runs in the device’s local storage.

The client is designed to accept all data that SedLOGIN can accept, plus some extra “field form” information that is not stored in NWIS but is useful for working records.

The first version of SedWE provides basic functionality, but does not have every feature we want or need. Some major elements that are NOT in Version 1.0 but we expect in future versions are:

  1. Support for non-USGS “Observer” users of the SedWE client
  2. An Image Module, for uploading photos from a site
  3. More ability to accept “field form” information that does not store in NWIS, but that is useful for working records and archiving with the site visit data.
